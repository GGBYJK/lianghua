
头肩顶与反向头肩顶评分机制
右肩确认 + 右肩/右颈半程触发版（100 分制）
一、适用目标
本评分机制用于已经完成五点结构识别，并在右肩确认后触发半程条件的头肩顶或反向头肩形态。评分必须在信号弹出的同一时刻完成，不等待颈线突破、突破放量、突破后延续或回踩确认。
信号生成与评分分离：满足形态硬条件和半程触发条件时立即发出信号并显示分数；分数用于衡量当前形态质量和排序，不用于把提醒延迟到颈线突破以后。

三、QTR 定义与统一用法
QTR 采用 N1 到 N2（包含两端）区间内真实波幅 TR 的平均值：
TR(i) = max[High(i) - Low(i), abs(High(i) - Close(i-1)), abs(Low(i) - Close(i-1))]
QTR = average(TR(i)), i = N1 ... N2
肩部差值 DS = abs(LS - RS)；颈线差值 DN = abs(N1 - N2)。肩部和颈线相似度统一按 DS/QTR、DN/QTR 分档，不再用固定点数或单纯百分比评分。QTR <= 0 或数据不足时，相似度项目记 0 分并标记数据异常。

7. 即时交易价值（10 分）
在触发 K 线 t0 上计算 neckline(t0)。形态高度 PH：头肩顶为 H - neckline(H)，反向头肩为 neckline(H) - H。
结构止损：头肩顶 Stop = RS + 0.25 QTR；反向头肩 Stop = RS - 0.25 QTR。
测量目标：头肩顶 Target = neckline(t0) - PH；反向头肩 Target = neckline(t0) + PH。
Risk = abs(TriggerPrice - Stop)；Reward = 沿预期方向从 TriggerPrice 到 Target 的距离；RR = Reward / Risk。若目标不在预期方向或 Risk <= 0，则 RR 项记 0 分。

八、程序化评分顺序
1. 识别 LS-N1-H-N2-RS，并检查形态硬条件。
2. 确认右肩拐点；从右肩后一根 K 线开始检查失效位和半程价。
3. 收盘价首次满足半程条件时，固定触发时点 t0。
4. 使用截至 t0 的数据计算 QTR、结构、颈线、时间、动能、触发质量和即时交易价值。
5. raw_score = trend + structure + neckline + time + momentum + trigger + trade_value。
6. final_score = min(raw_score, applicable_caps)，并随信号立即展示。
7. 颈线突破、突破放量、回踩和目标到达属于信号后的结果跟踪，可单独记录，不回填或篡改触发时刻的原始评分。
九、回测与参数校准建议
QTR 分档、0.25 QTR 止损缓冲、触发速度分档和评级阈值应分别按品种与周期回测。建议至少保存触发时分数、后续是否突破颈线、最大有利波动、最大不利波动和最终盈亏，用于校准各档分值，但不要在实时评分中引用未来结果。
